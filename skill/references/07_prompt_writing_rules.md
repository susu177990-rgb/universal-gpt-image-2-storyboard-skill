# 完整预制作导演板渲染规则

本文件只服务于 `cinematic_preproduction_board` 渲染器。

它不负责素材理解，也不负责导演规划。只有在 `preproduction_board_plan` 已经稳定后，才读取本文件。

## 渲染目标

输出一张完整的电影级导演预制作视觉板，质量至少要对齐专业提案板与影视预制作板。

## 渲染前提

必须已经拥有：

- `asset_lock_map`
- `preproduction_board_plan`
- 明确的 `output_purpose = review_or_pitch`

## 版面结构

### 区域 1：顶部概念区

包含：

- 标题
- subtitle
- logline
- tone tags
- production notes

### 区域 2：角色参考区

包含：

- 主角色身份
- 正侧背角度
- 表情特写
- 姿态参考
- 服装逻辑

### 区域 3：场景与美术区

包含：

- 场景主视图
- 关键细节图
- 生活化道具
- 色彩与材质氛围

### 区域 4：顶视机位阻挡区

包含：

- top view
- 人物站位
- 机位点位
- 摄影机方向
- 动线与关系线

### 区域 5：shot list 主区

包含：

- 按 shot 编号组织的主镜头
- 每镜头有明确视觉差异
- 每镜头承担独立导演功能

### 区域 6：底部技术与情绪区

包含：

- 灯光计划
- 摄影说明
- 色彩脚本
- 声音设计
- mood references

## 语言规则

- 全中文
- 用导演、摄影、美术、灯光语言
- 不用空泛形容词
- 信息丰富但不杂乱

## 提示词要求

最终提示词必须显式表达：

- 版面结构
- 每个大区块放什么
- 角色参考如何锁定
- 场景和美术如何呈现
- top view 与机位阻挡怎么表现
- shot list 如何组织
- 灯光、色彩、声音和 mood 如何收束

## 执行边界

- 本文件只定义完整预制作导演板提示词怎么写
- 不负责宣称“自动调用模型”
- 如果需要实际生图，必须由执行器层显式承担
