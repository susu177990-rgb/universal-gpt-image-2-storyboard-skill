# 设计报告契约

本文件定义当需要向用户解释设计思路时，应该如何输出。

只解释与当前用户输入直接相关的判断，不写空泛分析。

## 报告结构

1. **用户目标理解**
   - 当前最核心的表演重点是什么

2. **素材锁定图**
   - 每个素材负责什么
   - 每个素材禁止继承什么

3. **分镜类型判断**
   - 为什么选这个 `board_type`

4. **格数判断**
   - 为什么是这个 `panel_count`

5. **每格功能规划**
   - 每格在叙事和导演控制上承担什么作用

6. **连续性风险**
   - 最容易翻车的地方是什么
   - 如何在提示词中规避

7. **最终输出方向**
   - 为什么选择 6 区提案板或纯净参考板
