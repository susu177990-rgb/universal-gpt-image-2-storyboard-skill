# 示例 05：产品交互请求（发光血清）

## 输入

- 输入模式：素材与文本混合
- 输出目的：给人审核 / 提案
- 主动作：女人拿起发光血清并凑近观察
- 素材：
  - `woman_ref`：角色
  - `serum_ref`：产品

## 中间结果重点

### `asset_lock_map`

- `woman_ref` 锁定角色身份
- `serum_ref` 锁定产品外形、比例、绿色发光特征

### `storyboard_plan`

- 分镜类型：`product_interaction_board`
- 格数：4
- 风险控制：产品必须全程可见，不能被角色手掌完全遮挡

## 预期输出

### Markdown 提示词片段

````md
# 发光血清观察

## 导演规划摘要

- 分镜类型：`product_interaction_board`
- 重点：角色注意力向产品转移，最后用特写展示产品特征

## 最终渲染提示词

```text
请生成一张全中文 6 区故事板提案板。
主分镜 4 格依次展示：人物与产品关系建立、手部接近、拿起接触、产品特写展示。
保持产品比例正确，玻璃材质与绿色发光稳定。
```
````
