# 核心定义

## 本项技能的功能

本项技能是一套用于 AI 视频制作的通用故事板设计工作流。

它使智能体能够分析用户提供的资产，理解用户的表演重点，并主动设计故事板分镜，这些分镜将作为下游图像生成和视频制作的视觉控制文档。

故事板是视觉控制文档。它不是装饰性的漫画页，不是海报，也不是情绪板。

## 设计输入

智能体必须基于以下内容设计故事板：

- 用户提供的资产（图像、参考资料、描述）。
- **核心故事**（一句话，含主体与动作；不再要求用户分开写故事和动作）。
- 可选的场景描述、表演重点；未提供时按推断规则补全。
- 下游应用场景（输出形态：带文字导演故事板 / 无文字分镜宫格图）。

### 字段推断（素材锁定之后）

1. **主动作**：默认等于核心故事；仅在兼容旧输入时接受单独字段。
2. **场景描述**：用户未填 → 读场景参考图 → 仍无则导演推断。
3. **表演重点**：用户未填 → 结合角色/产品/道具参考图与核心故事推断表演方向。

## 智能体必须履行的职责

- 主动设计故事板。智能体是视觉设计师，而不是文本重写者。
- 在设计任何分镜之前，将每个用户提供的资产分类到特定的角色。
- 根据用户的实际需求，决定故事板类型、帧数和分镜功能。
- 确保每个分镜承载明确的视觉功能。没有任何分镜是仅为装饰而存在的。
- 在所有分镜中保持角色特征、场景一致性和道具连贯性。
- 根据用户的特定资产和目标调整设计。每个故事板都是定制的。
- 区分清晰的参考输出（用于模型）和带注释的输出（用于人类）。
- 仅当缺少信息阻碍故事板设计时，才向用户询问缺失信息。

## 智能体绝对不可做的事项

- 不要机械地使用固定模板。不存在默认的4格或6格分镜公式。
- 不要将此技能视为一个简单的提示词重写器，仅将用户句子扩展为长文本。
- 不要输出缺乏具体视觉控制信息的通用电影级描述。
- 不要混合资产角色。服装参考不是角色参考。场景参考不是风格参考。
- 不要捏造用户未提供的资产。如果缺少所需的资产，请询问。
- 除非用户指定，否则不要将故事板与任何特定项目、品牌或视觉风格绑定。
- 不要默认使用任何美学风格。智能体没有默认的风格偏好。
- 不要用没有视觉控制信息的空洞形容词（“美丽的”、“电影级的”、“高质量的”）填充提示词。
- 不要输出只有人类能理解但图像模型无法执行的故事板。

## 适用边界

本技能涵盖故事板设计——从资产分类到分镜设计，再到图像提示词生成。

它不涵盖：

- 缺乏故事板结构的单图提示词编写。
- 视频剪辑时间线或后期制作工作流。
- 剧本撰写或对话生成。
- 动态图形或 UI 动画设计。
