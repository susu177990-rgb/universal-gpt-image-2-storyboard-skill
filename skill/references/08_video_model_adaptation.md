# 输出形态与模型适配

本文件定义渲染器选择规则。

## 输出形态

### 1. 完整预制作导演板

适用场景：

- 给人审核
- 给客户提案
- 给导演、美术、摄影、制片对齐

特征：

- 有概念区
- 有角色参考区
- 有场景与美术区
- 有顶视机位阻挡区
- 有 shot list 主区
- 有技术与情绪区

### 2. 纯净参考板

适用场景：

- 图像转视频
- 下游视觉模型参考
- 不希望文字和 UI 干扰模型理解

特征：

- 只有图像分镜
- 没有头栏和尾栏
- 没有说明文字、编号框、箭头、对话框

## 选择规则

- 用户明确说“给人看、汇报、提案” -> 完整预制作导演板
- 用户明确说“给模型吃、给视频模型参考” -> 纯净参考板
- 用户未说明但更像人工沟通材料 -> 默认完整预制作导演板

## 重要原则

完整预制作导演板是默认渲染器。

真正的核心不是版式，而是：

- 素材锁定是否准确
- 导演预制作信息是否完整
- 导演规划是否清晰
- 连续性规则是否完整

## 纯净参考板限制

一旦进入纯净参考板模式，必须去掉：

- 标题文字
- 面板编号
- caption 文案
- UI 样式边框
- 技术参数说明

让画面本身承担叙事信息。
