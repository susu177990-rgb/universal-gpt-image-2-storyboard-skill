# 输入要求与表单边界

本文件是输入契约的中文说明，与 `interface/input.json` 保持一致。

## 最小必填信息

以下信息缺一不可：

1. **核心故事**
   - 用户想让镜头里发生什么
2. **主体与动作**
   - 谁或什么在执行动作
3. **场景信息**
   - 动作发生在哪里
4. **素材列表**
   - 如果是素材驱动或混合模式，至少提供一项素材
   - 如果是纯文本创作，必须明确声明输入模式
5. **输出目的**
   - 给人看、给模型吃、还是直接生图

## 推荐输入字段

### 1. 项目信息

- 标题
- 输入模式
- 输出目的
- 输出模式
- 输出语言

### 2. 故事请求

- 核心故事
- 主动作
- 场景描述
- 视觉目标

### 3. 素材列表

每个素材推荐包含：

- `asset_id`
- `role_tag`
- `asset_url`
- `description`
- `must_keep`
- `must_avoid`

### 4. 可选参数

- 宽高比
- 分镜类型偏好
- 镜头数偏好
- 时长参考
- 机位偏好
- 风格目标
- 是否存在上一帧
- 是否允许模型自动补全次要细节

## 允许推断的内容

以下内容可以在上下文足够时自动推断：

- 用户没指定分镜类型，但动作结构明显
- 用户没指定画幅，但输出目标明确
- 用户没指定风格，但提供了明确风格素材
- 用户没指定镜头数，但动作复杂度足够判断

## 必须补问的内容

以下情况必须补问：

- 只有素材，没有动作目标
- 有动作，但主体身份完全不清
- 有多个素材，但角色分工根本无法判断
- 输出目的不明，导致无法确定渲染器

## 补问规则

- 每次最多补问 1 到 2 项
- 只问阻塞项，不问好奇项
- 能推断就不问
- 问题必须是全中文、短句、可直接回答

## 补问格式

```text
现在还不能继续，缺少：____。
请补充：____。
补充后我会直接继续生成故事板。
```
