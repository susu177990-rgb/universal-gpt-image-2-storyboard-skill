# 设计报告契约

本文件定义当需要向用户解释设计思路时，应该如何输出。

只解释与当前用户输入直接相关的判断，不写空泛分析。

## 报告结构

1. **用户目标理解**
   - 当前最核心的故事目标、表演重点和关系动态是什么

2. **素材锁定图**
   - 每个素材负责什么
   - 每个素材禁止继承什么
   - 每个素材提供了哪些导演锚点

3. **导演板类型判断**
   - 为什么选这个 `board_type`
   - 为什么这个输出更适合完整预制作板而不是普通分镜

4. **shot list 判断**
   - 为什么是这个镜头数量
   - 为什么这样组织镜头语言和节奏

5. **blocking 与视觉规划**
   - 顶视机位与动线如何组织
   - 场景、美术、灯光和人物关系如何协同

6. **连续性风险**
   - 最容易翻车的地方是什么
   - 如何在提示词中规避

7. **最终输出方向**
   - 为什么选择完整预制作导演板或纯净参考板
